# CC6.2: Registration and authorization of new users

## Purpose

Prior to issuing system credentials and granting system access, the entity registers and
authorizes new internal and external users.

## Description

The organization has a formal process for onboarding users that includes identity
verification, authorization by an appropriate manager, assignment of access based on
the principle of least privilege, and documentation of the access granted.

## Key Considerations

- Formal user registration and de-registration process
- Identity verification prior to access provisioning
- Manager authorization required before access is granted
- Least-privilege principle applied
- Access provisioning is documented and auditable

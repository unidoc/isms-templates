# CC7.1: Detection and monitoring of anomalies

## Purpose

To meet its objectives, the entity uses detection and monitoring procedures to identify
changes to configurations that result in the introduction of new vulnerabilities, and
susceptibilities to newly discovered vulnerabilities.

## Description

The organization implements security monitoring tools and processes to detect anomalies
that may indicate security events. This includes SIEM systems, log aggregation,
configuration monitoring, vulnerability scanning, and threat intelligence feeds.

## Key Considerations

- SIEM or log management system is deployed and configured
- Security events are correlated and analysed
- Vulnerability scanning is performed regularly
- Configuration changes are monitored and validated
- Threat intelligence feeds inform detection capabilities

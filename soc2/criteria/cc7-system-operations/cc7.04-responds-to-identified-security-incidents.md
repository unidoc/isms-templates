# CC7.4: Incident response

## Purpose

The entity responds to identified security incidents by executing a defined incident
response programme to understand, contain, remediate, and communicate security incidents.

## Description

The organization maintains an incident response plan that defines roles, responsibilities,
communication protocols, containment strategies, eradication procedures, recovery steps,
and post-incident review processes. The plan is tested regularly through tabletop exercises
or simulations.

## Key Considerations

- Incident response plan is documented and approved
- Incident response team with defined roles and responsibilities
- Containment, eradication, and recovery procedures are defined
- Communication protocols cover internal and external notifications
- Post-incident reviews are conducted and lessons learned are documented
- Plan is tested at least annually

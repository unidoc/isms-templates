# CC7.2: Monitoring of system components for anomalies

## Purpose

The entity monitors system components and the operation of those components for anomalies
that are indicative of malicious acts, natural disasters, and errors affecting the entity's
ability to meet its objectives. Anomalies are analyzed to determine whether they represent
security events.

## Description

The organization continuously monitors infrastructure, applications, and network components
for anomalous behaviour. Monitoring covers system performance, security alerts, access
patterns, and data flows. Identified anomalies are triaged and investigated.

## Key Considerations

- Infrastructure monitoring covers servers, networks, and applications
- Baseline behaviour is established for anomaly detection
- Alerts are triaged and investigated by qualified personnel
- Monitoring covers both on-premises and cloud environments
- Monitoring is 24/7 or as appropriate to risk

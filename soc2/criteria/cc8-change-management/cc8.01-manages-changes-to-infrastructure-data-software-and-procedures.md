# CC8.1: Change management

## Purpose

The entity authorizes, designs, develops or acquires, configures, documents, tests,
approves, and implements changes to infrastructure, data, software, and procedures to
meet its objectives.

## Description

The organization implements a formal change management process that covers all changes
to the production environment, including infrastructure, applications, databases, and
procedures. Changes are classified, risk-assessed, tested, approved, and documented
before implementation. Emergency change procedures are defined for urgent situations.

## Key Considerations

- Change management policy and procedures are documented
- Changes are classified by type and risk level
- Impact and risk assessment is performed before implementation
- Testing and validation are required before production deployment
- Changes require appropriate approval
- Emergency change procedures are defined
- Change records are maintained for audit purposes
